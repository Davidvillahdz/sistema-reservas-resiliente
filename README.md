\# Sistema de Reservas Resiliente



Práctica de tolerancia a fallos sobre Kubernetes.



\## Integrantes



\- David Villa Hernández

\- Juan Álvarez



\## Objetivo



Construir y desplegar un sistema distribuido de venta de entradas, compuesto por:



\- API Gateway

\- Servicio de Reservas

\- Servicio de Inventario

\- Servicio de Pagos

\- Servicio de Notificaciones

\- Base de Datos



El sistema será desplegado en un clúster Kubernetes de al menos dos nodos y será sometido a escenarios controlados de fallo.



\## Tecnologías



\- Python

\- FastAPI

\- PostgreSQL

\- Docker

\- Kubernetes

\- kind

\- k6
